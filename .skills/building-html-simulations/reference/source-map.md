# Source Map

Use this file to refresh or cite the ideas behind the skill. Prefer official docs for APIs and primary examples for design patterns.

## Web Platform

- MDN Canvas API: https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API
  - Use for Canvas 2D drawing, animation, game graphics, data visualization, image manipulation, and real-time rendering.

- MDN `requestAnimationFrame`: https://developer.mozilla.org/en-US/docs/Web/API/Window/requestAnimationFrame
  - Use for browser-scheduled animation callbacks before repaint. Remember it is callback-based and should be requested again for continuing animation.

- MDN WebGL API: https://developer.mozilla.org/en-US/docs/Web/API/WebGL_API
  - Use for high-performance interactive 2D/3D graphics in a canvas with hardware acceleration.

- MDN WebGPU API: https://developer.mozilla.org/en-US/docs/Web/API/WebGPU_API
  - Use only when current browser support and project complexity justify a lower-level modern GPU API.

## 3D and Physics

- Three.js docs: https://threejs.org/docs/
  - Use for scene, camera, renderer, materials, lights, controls, loaders, helpers, and addon APIs.

- Three.js examples: https://threejs.org/examples/
  - Use for implementation patterns and minimal examples when building a specific effect or interaction.

- Babylon.js docs: https://doc.babylonjs.com/
  - Use when a full-featured web rendering engine is a better fit than assembling lower-level Three.js pieces.

- Matter.js docs: https://brm.io/matter-js/docs/
  - Use for browser-based 2D rigid bodies, constraints, collisions, mouse constraints, engine, runner, and render modules.

- Rapier JS docs: https://rapier.rs/docs/user_guides/javascript/getting_started_js/
  - Use for fast 2D/3D physics in JavaScript via WebAssembly. Account for asynchronous loading.

## Creative Coding and Data

- p5.js: https://p5js.org/
  - Use for approachable creative coding, educational sketches, and fast visual prototypes.

- p5.js reference: https://p5js.org/reference/
  - Use for drawing, interaction, animation, sound, media, and sketch APIs.

- D3: https://d3js.org/
  - Use for custom dynamic data visualization with SVG, Canvas, HTML, scales, axes, layouts, and interaction.

- Observable notebooks docs: https://observablehq.com/documentation/notebooks/
  - Use as a pattern for reactive parameters, code, Markdown explanation, and outputs in one interactive document.
